SKU Price Compare

So sánh file Excel giá xưởng với bảng sku_price trên PostgreSQL

1. Khóa tìm bản ghi DB

Mỗi dòng Excel được tìm trong DB theo khóa (giống unique index):

factory + sku_factory + type + print_tech + include + DTG Print SKU

factory lấy từ tên sheet Excel (vd: LK401). Các trường còn lại lấy từ cột tương ứng trên dòng Excel. DTG Print SKU tra từ sheet SKU MAPPING (xem mục 4) — phía DB dùng cột sku_ecb.

2. Phân loại kết quả

Bảng / trạng thái Điều kiện
Không có trong DB Không tra được DTG Print SKU từ SKU MAPPING, hoặc không tìm thấy bản ghi DB với cùng khóa 6 trường
Khớp Tìm thấy bản ghi DB và mọi trường so sánh đều bằng nhau
Sai khác Tìm thấy bản ghi DB nhưng có ít nhất 1 trường khác nhau
Chỉ có trong DB Bản ghi DB thuộc xưởng đang so sánh nhưng không có dòng Excel nào khớp khóa
Dòng Excel → tra DTG Print SKU từ SKU MAPPING → không có → Không có trong DB
có → tìm DB theo khóa 6 trường → không có → Không có trong DB
có → so từng trường → tất cả bằng → Khớp / có khác → Sai khác
Bản ghi DB cùng xưởng → key không xuất hiện trong Excel → Chỉ có trong DB

3. Quy tắc so sánh từng trường

So các cột trên sheet xưởng (trừ ID): SKU, Product Name, Color, Size, Type, PrintTech, Include, Exclude, Tag, Stock, Price, Flash sale 1, Flash sale 2, Additional Price.

Loại trường Cách so
Stock So boolean — Y, yes, true, 1 = có stock
Price, Flash sale 1/2, Additional Price So số; chênh lệch ≤ 0.005 coi là bằng; null = null thì bằng
Include, Exclude Trim chuỗi; chuỗi rỗng vẫn là rỗng
Các trường còn lại Trim chuỗi; chuỗi rỗng coi như null

4. Tra DTG Print SKU (sheet SKU MAPPING)

Sheet SKU MAPPING (dòng 1 = header) có cột DTG Print SKU và các cột xưởng (NB1, LK401, …).

Lấy ID của dòng trên sheet xưởng → tìm dòng trong SKU MAPPING có giá trị cột xưởng tương ứng = ID đó → lấy DTG Print SKU (có thể nhiều dòng cùng ID) → thử từng giá trị cho đến khi khớp sku_ecb trong DB

Không tìm thấy mapping → vào bảng «Không có trong DB».

Cùng SKU + type + printTech nhưng include khác → coi là 2 bản ghi khác nhau. Dòng Excel bị bỏ qua khi parse (thiếu SKU, v.v.) không vào 4 bảng trên — chỉ đếm ở cột «Bỏ qua».
